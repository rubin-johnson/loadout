## BT-001 — Apply/restore roundtrip leaves target byte-identical

### User story
As a developer running automated tests, I want the apply-then-restore cycle to leave the target directory byte-for-byte identical to its pre-apply state so that `loadout restore` is a genuine undo.

### Context
SC-2: `loadout apply ./bundle --yes` followed by `loadout restore --yes` must leave the target identical. This cross-cuts STORY-007 (apply) and STORY-008 (restore) and cannot be expressed in either story's unit tests alone.

### Acceptance criteria
1. After apply + restore on a temp target dir, every file present before apply is present after restore with identical content.
2. No files introduced by the bundle remain in the target after restore.
3. Test passes without touching `~/.claude/`.

### Unit tests (in this story)
```python
# tests/test_bt.py
import pathlib, shutil, subprocess, sys, pytest

def run(*args, **kwargs):
    return subprocess.run(
        [sys.executable, "-m", "loadout", *args],
        capture_output=True, text=True, **kwargs
    )

def _dir_checksums(path: pathlib.Path) -> dict:
    result = {}
    for f in sorted(path.rglob("*")):
        if f.is_file() and ".loadout-" not in str(f):
            result[str(f.relative_to(path))] = f.read_bytes()
    return result

def test_bt001_roundtrip(tmp_path):
    # build a minimal bundle
    bundle = tmp_path / "bundle"
    bundle.mkdir()
    (bundle / "manifest.yaml").write_text(
        "name: test\nversion: 0.1.0\nauthor: ci\ndescription: d\n"
        "targets:\n  - path: CLAUDE.md\n    dest: CLAUDE.md\n"
    )
    (bundle / "CLAUDE.md").write_text("# test")

    target = tmp_path / "target"
    target.mkdir()
    (target / "CLAUDE.md").write_text("# original")
    (target / "extra.txt").write_text("keep me")

    before = _dir_checksums(target)

    r = run("apply", str(bundle), "--target", str(target), "--yes")
    assert r.returncode == 0, r.stderr

    r = run("restore", "--target", str(target), "--yes")
    assert r.returncode == 0, r.stderr

    after = _dir_checksums(target)
    assert before == after
```

## BT-002 — Partial failure does not corrupt target

### User story
As a user, I want a mid-apply failure to leave my target directory unchanged so that a failed apply cannot produce a broken Claude environment.

### Context
SC-4 / R-08: `apply` uses a temp-dir-then-rename strategy. A simulated failure mid-operation must not corrupt the target.

### Acceptance criteria
1. When the rename step is mocked to raise `OSError` after temp dir is populated, the target directory matches its pre-apply state.
2. `apply` exits non-zero on the simulated failure.

### Unit tests (in this story)
```python
# tests/test_bt.py (continued)
import pathlib, pytest
from unittest.mock import patch

def test_bt002_partial_failure_atomic(tmp_path):
    from loadout.apply import atomic_apply
    from loadout.manifest import load_manifest

    bundle = tmp_path / "bundle"
    bundle.mkdir()
    (bundle / "manifest.yaml").write_text(
        "name: test\nversion: 0.1.0\nauthor: ci\ndescription: d\n"
        "targets:\n  - path: a.txt\n    dest: a.txt\n"
    )
    (bundle / "a.txt").write_text("new")

    target = tmp_path / "target"
    target.mkdir()
    (target / "a.txt").write_text("original")

    manifest = load_manifest(bundle)

    with patch("loadout.apply.shutil.move", side_effect=OSError("disk full")):
        with pytest.raises(OSError):
            atomic_apply(bundle, target, manifest)

    assert (target / "a.txt").read_text() == "original"
```

### Implementation notes
- `atomic_apply` writes to a temp dir inside `target`'s parent, then calls `shutil.move`. Patching `shutil.move` at its use site simulates failure after temp population but before commit.

## STORY-002 — Manifest model and parsing

### User story
As a developer, I want a `Manifest` dataclass and `load_manifest()` function so that every command that reads a bundle has a single, validated entry point for manifest data.

### Context
R-01 through R-04 define the bundle format and manifest schema. Every command that touches a bundle calls `load_manifest()`. Semver validation is part of R-25.

### Acceptance criteria
1. `load_manifest(bundle_dir)` returns a `Manifest` with all five required fields populated.
2. Missing any required field raises `ManifestError` with the field name in the message.
3. A non-semver `version` string raises `ManifestError` mentioning "version" or "semver".
4. `manifest.yaml` not found raises `ManifestError` mentioning "manifest.yaml".
5. Each `targets` entry is a `TargetEntry` with `path` and `dest` attributes.

### Unit tests (in this story)
```python
# tests/test_manifest.py
import pytest, yaml, pathlib
from loadout.manifest import Manifest, TargetEntry, ManifestError, load_manifest

def _write_manifest(path: pathlib.Path, data: dict):
    (path / "manifest.yaml").write_text(yaml.dump(data))

VALID = {
    "name": "my-loadout", "version": "1.2.3", "author": "me",
    "description": "desc",
    "targets": [{"path": "CLAUDE.md", "dest": "~/.claude/CLAUDE.md"}]
}

def test_load_valid(tmp_path):
    _write_manifest(tmp_path, VALID)
    m = load_manifest(tmp_path)
    assert m.name == "my-loadout"
    assert m.version == "1.2.3"
    assert len(m.targets) == 1
    assert m.targets[0].path == "CLAUDE.md"
    assert m.targets[0].dest == "~/.claude/CLAUDE.md"

def test_missing_manifest_file(tmp_path):
    with pytest.raises(ManifestError, match="manifest.yaml"):
        load_manifest(tmp_path)

@pytest.mark.parametrize("field", ["name", "version", "author", "description", "targets"])
def test_missing_required_field(tmp_path, field):
    data = {k: v for k, v in VALID.items() if k != field}
    _write_manifest(tmp_path, data)
    with pytest.raises(ManifestError, match=field):
        load_manifest(tmp_path)

@pytest.mark.parametrize("bad", ["1.0", "v1.0.0", "not-semver", "1.2.3.4", ""])
def test_invalid_semver(tmp_path, bad):
    data = {**VALID, "version": bad}
    _write_manifest(tmp_path, data)
    with pytest.raises(ManifestError):
        load_manifest(tmp_path)

@pytest.mark.parametrize("good", ["0.0.1", "1.0.0", "1.2.3-alpha.1", "2.0.0+build.1"])
def test_valid_semver(tmp_path, good):
    data = {**VALID, "version": good}
    _write_manifest(tmp_path, data)
    m = load_manifest(tmp_path)
    assert m.version == good
```

### Implementation notes
- `loadout/manifest.py`: define `ManifestError(ValueError)`, `TargetEntry` (dataclass with `path: str`, `dest: str`), `Manifest` (dataclass with `name`, `version`, `author`, `description`, `targets: list[TargetEntry]`).
- Semver regex: `^(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)(?:-[0-9A-Za-z-]+(?:\.[0-9A-Za-z-]+)*)?(?:\+[0-9A-Za-z-]+(?:\.[0-9A-Za-z-]+)*)?$` — compile once at module level.
- `load_manifest(bundle_dir: pathlib.Path) -> Manifest`: read YAML, validate each field, return dataclass. Use `yaml` from stdlib-available `PyYAML`. Note: `yaml` is not stdlib, but it's unavoidable for YAML parsing. Since R-30 says "zero non-stdlib runtime dependencies" — use `tomllib` (stdlib 3.11) if TOML is acceptable, or use `yaml` (PyYAML). Given the PRD explicitly uses YAML, add `pyyaml` as a runtime dep. This is the one justified exception.
- Add `pyyaml` to `pyproject.toml` dependencies.

## STORY-003 — `loadout validate` command

### User story
As a bundle author, I want `loadout validate <bundle>` to check every conformance rule and report all failures so that I can fix my bundle before distributing it.

### Context
R-04 / R-25 / R-26. `validate` is also called internally by `apply` (R-07). It must be usable both as a standalone CLI command and as a library function returning structured errors.

### Acceptance criteria
1. `validate_bundle(bundle_dir)` returns `list[str]` — empty on success, one error string per failure.
2. Missing `manifest.yaml` → one error mentioning "manifest.yaml".
3. Invalid semver → one error.
4. Each `targets` entry whose `path` doesn't exist in the bundle → one error per missing file.
5. CLI `loadout validate <dir>` exits 0 on valid bundle; exits 1 with one error per line on failure.
6. Multiple errors are all reported (not fail-fast).

### Unit tests (in this story)
```python
# tests/test_validate.py
import pathlib, yaml, pytest
from loadout.validate import validate_bundle

def _bundle(tmp_path, **overrides):
    d = tmp_path / "bundle"
    d.mkdir()
    data = {
        "name": "t", "version": "0.1.0", "author": "a",
        "description": "d",
        "targets": [{"path": "CLAUDE.md", "dest": "CLAUDE.md"}]
    }
    data.update(overrides)
    (d / "CLAUDE.md").write_text("x")
    (d / "manifest.yaml").write_text(yaml.dump(data))
    return d

def test_valid_bundle_returns_empty(tmp_path):
    b = _bundle(tmp_path)
    assert validate_bundle(b) == []

def test_missing_manifest(tmp_path):
    d = tmp_path / "bundle"
    d.mkdir()
    errors = validate_bundle(d)
    assert len(errors) == 1
    assert "manifest.yaml" in errors[0]

def test_missing_target_file(tmp_path):
    b = _bundle(tmp_path)
    (b / "CLAUDE.md").unlink()
    errors = validate_bundle(b)
    assert any("CLAUDE.md" in e for e in errors)

def test_invalid_semver(tmp_path):
    b = _bundle(tmp_path, version="bad")
    errors = validate_bundle(b)
    assert len(errors) >= 1

def test_multiple_errors_reported(tmp_path):
    b = _bundle(tmp_path, version="bad")
    (b / "CLAUDE.md").unlink()
    errors = validate_bundle(b)
    assert len(errors) >= 2

def test_targets_missing_dest_field(tmp_path):
    b = _bundle(tmp_path)
    (b / "manifest.yaml").write_text(yaml.dump({
        "name": "t", "version": "0.1.0", "author": "a", "description": "d",
        "targets": [{"path": "CLAUDE.md"}]  # no dest
    }))
    errors = validate_bundle(b)
    assert any("dest" in e for e in errors)
```

- `loadout/validate.py`: `def validate_bundle(bundle_dir: pathlib.Path) -> list[str]`. Accumulate errors in a list; never raise. First check for `manifest.yaml` existence before attempting parse. Catch `ManifestError` from `load_manifest` and append its message. Then separately check each target's `path` exists.
- `cli.py` `validate` handler: call `validate_bundle`, if errors print each to stdout and `sys.exit(1)`, else print "OK" and exit 0.

## STORY-006 — Atomic apply (file write engine)

### User story
As a developer, I want `atomic_apply(bundle_dir, target_dir, manifest)` to write bundle files to the target using a temp-dir-then-rename strategy so that partial failures leave the target unchanged.

### Context
R-08: temp-dir-then-rename. The `targets` list in the manifest maps bundle-relative paths to destination paths relative to `target_dir` (after `~` expansion). Only files listed in `targets` are written.

### Acceptance criteria
1. After `atomic_apply`, each file in `targets` exists at `target_dir / target_entry.dest` (with `~` expanded relative to `target_dir`, not the real home).
2. Files already in `target_dir` but not in `targets` are left untouched.
3. If `shutil.move` raises, the target directory is unmodified.
4. Nested destination paths (`hooks/pre-apply.sh`) are created with `mkdir -p`.
5. `dest` paths with `~/` prefix are resolved relative to `target_dir` (i.e., `~/` = `target_dir/`).

### Unit tests (in this story)
```python
# tests/test_apply.py
import pathlib, shutil, pytest, yaml
from unittest.mock import patch
from loadout.manifest import load_manifest
from loadout.apply import atomic_apply

def _make_bundle(tmp_path, files: dict, targets_override=None):
    bundle = tmp_path / "bundle"
    bundle.mkdir()
    for name, content in files.items():
        p = bundle / name
        p.parent.mkdir(parents=True, exist_ok=True)
        p.write_text(content)
    targets = targets_override or [
        {"path": name, "dest": name} for name in files
    ]
    (bundle / "manifest.yaml").write_text(yaml.dump({
        "name": "t", "version": "0.1.0", "author": "a",
        "description": "d", "targets": targets
    }))
    return bundle

def test_files_land_at_correct_paths(tmp_path):
    bundle = _make_bundle(tmp_path, {"CLAUDE.md": "new"})
    target = tmp_path / "target"
    target.mkdir()
    manifest = load_manifest(bundle)
    atomic_apply(bundle, target, manifest)
    assert (target / "CLAUDE.md").read_text() == "new"

def test_unrelated_files_untouched(tmp_path):
    bundle = _make_bundle(tmp_path, {"CLAUDE.md": "new"})
    target = tmp_path / "target"
    target.mkdir()
    (target / "unrelated.txt").write_text("keep")
    manifest = load_manifest(bundle)
    atomic_apply(bundle, target, manifest)
    assert (target / "unrelated.txt").read_text() == "keep"

def test_nested_dest_created(tmp_path):
    targets = [{"path": "hook.sh", "dest": "hooks/hook.sh"}]
    bundle = _make_bundle(tmp_path, {"hook.sh": "#!/bin/bash"}, targets)
    target = tmp_path / "target"
    target.mkdir()
    manifest = load_manifest(bundle)
    atomic_apply(bundle, target, manifest)
    assert (target / "hooks" / "hook.sh").read_text() == "#!/bin/bash"

def test_tilde_dest_resolves_to_target(tmp_path):
    targets = [{"path": "CLAUDE.md", "dest": "~/CLAUDE.md"}]
    bundle = _make_bundle(tmp_path, {"CLAUDE.md": "tilde"}, targets)
    target = tmp_path / "target"
    target.mkdir()
    manifest = load_manifest(bundle)
    atomic_apply(bundle, target, manifest)
    assert (target / "CLAUDE.md").read_text() == "tilde"

def test_atomic_on_move_failure(tmp_path):
    bundle = _make_bundle(tmp_path, {"a.txt": "new"})
    target = tmp_path / "target"
    target.mkdir()
    (target / "a.txt").write_text("original")
    manifest = load_manifest(bundle)
    with patch("loadout.apply.shutil.move", side_effect=OSError("fail")):
        with pytest.raises(OSError):
            atomic_apply(bundle, target, manifest)
    assert (target / "a.txt").read_text() == "original"
```

### Implementation notes
- `loadout/apply.py`: `def atomic_apply(bundle_dir, target_dir, manifest)`.
- Strategy: create `tempfile.mkdtemp(dir=target_dir.parent)` as staging area. Copy all target files into staging. Then `shutil.move(staging, final_staging)` and swap individual files from staging to target. Actually simpler: copy only the bundle's target files into a temp dir structured like the target, then for each file `shutil.move(tmp_file, target_file)`. The key atomicity guarantee is that each individual file move is atomic on the same filesystem (rename syscall). If any move fails, already-moved files are moved, but the error surfaces. For true all-or-nothing: copy files to a temp dir tree, then do all moves. If a move fails partway, roll back the already-moved files from the temp dir. Implement a simple rollback: keep a `moved: list[tuple[src_tmp, dest_actual]]` and on exception move them back.
- `dest` resolution: `if dest.startswith("~/") or dest == "~"`: strip `~/` prefix, resolve relative to `target_dir`. Otherwise resolve relative to `target_dir`.

## STORY-011 — `loadout status` command

### User story
As a user, I want `loadout status [--target <dir>]` to show me the currently applied loadout so that I know what configuration is active without inspecting files manually.

### Context
R-27 / R-28. Reads `.loadout-state.json`. Exits 0 in all cases (including no state file).

### Acceptance criteria
1. With a state file present, prints `active`, `manifest_version`, `applied_at`, `bundle_path`.
2. Without a state file, prints "No loadout currently applied" and exits 0.
3. Respects `--target <dir>` and `LOADOUT_TARGET_ROOT`.
4. Output is human-readable (not raw JSON).

### Unit tests (in this story)
